2Curved connectors + avatar markers + hover/selected states + tags

Riffing on 1a and 1c (now the only two concepts — the relaxed "Commit Cards" direction is dropped, everything is compact): branch connectors are smooth S-curves instead of right angles, every commit marker on the line is an avatar circle (initials computed from the author when there's no photo), rows are interactive — hover to preview, click to select — and tags now show up as a small notched, neutral-grey chip next to the branch pill (see the "main" row, tagged v2.4.0) — deliberately un-colored so they read as a point-in-time label rather than competing with the branch color coding.

1Git Wasp — graph view concepts

Two structural takes on the commit-graph pane, deliberately away from GitKraken's dense free-form bezier layout: fixed parallel swim-lanes per branch, orthogonal (right-angle) connectors instead of smooth curves. Same compact density, same 13-commit sample history (6 branches live at once, two merges, a working-changes row, a tagged release, and the current HEAD marked) — only the column layout changes.

1aLedger Grid
True spreadsheet columns — graph, commit, author, branch, hash and date each get their own aligned column. Compact, dark, built for scanning a lot of history fast. See the curved, interactive version at 2a.
